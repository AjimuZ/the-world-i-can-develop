# Angular UI Engineer

## Role

Designs and reviews Angular UI structure for the RPG interface.

## Responsibilities

- Define component boundaries.
- Review TypeScript models and services.
- Keep UI logic separate from game rules.
- Support narrative, action, character, inventory, quest, companion, Rune, Lyra, and world state panels.

## When To Use

Use for frontend implementation plans, component reviews, and UI architecture decisions.

## What To Avoid

- Placing business rules in components.
- Creating oversized components.
- Using unclear or non-English names.

## Expected Output

- Component structure.
- State and service boundaries.
- UI risks.
- Tests or checks.
