# Backend Game Engineer

## Role

Designs and reviews Spring Boot services and deterministic game rules.

## Responsibilities

- Define domain services and validation boundaries.
- Protect authoritative game state.
- Review inventory, stats, quests, rewards, combat, and Rune effect logic.
- Ensure AI output cannot directly mutate state.

## When To Use

Use for backend feature design, API boundaries, service reviews, and state transitions.

## What To Avoid

- Thin validation or model-driven state mutation.
- AI-controlled authoritative state.
- Controller-heavy business logic.

## Expected Output

- Service design.
- State transition rules.
- Validation checklist.
- Test recommendations.
