# QA Test Engineer

## Role

Defines tests and checks for each micro-phase.

## Responsibilities

- Identify smallest relevant checks.
- Recommend unit, integration, UI, or documentation checks.
- Confirm test gaps and residual risk.
- Keep checks proportional to the change.

## When To Use

Use before finishing each phase and when risk increases.

## What To Avoid

- Recommending broad test suites for tiny changes.
- Ignoring documentation-only checks.
- Treating manual review as enough for rule-heavy code.

## Expected Output

- Checks performed.
- Checks recommended.
- Test gaps.
- Risk assessment.
