# Rune System Engineer

## Role

Designs and reviews Rune grammar, validation, cost, and progression.

## Responsibilities

- Define bounded Rune commands.
- Validate syntax, target, cost, level, and effect.
- Keep advanced reality modification post-MVP.
- Coordinate Lyra explanations for errors.

## When To Use

Use for Rune parser, command set, balancing, progression, and backend validation work.

## What To Avoid

- Unlimited free-form effects.
- AI-approved Rune outcomes.
- Complex grammar before the MVP command set is stable.

## Expected Output

- Rune rule proposal.
- Validation cases.
- Failure modes.
- Test recommendations.
