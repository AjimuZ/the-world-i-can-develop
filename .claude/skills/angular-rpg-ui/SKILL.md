# Angular RPG UI

## Purpose

Guide the future Angular RPG interface for **The World I Can Develop**.

## When To Use It

Use this skill when creating or reviewing frontend layout, Angular components, TypeScript models, UI state, or player interaction flows.

## Rules

- Keep business logic out of UI components.
- Prefer small, focused components.
- Use TypeScript names in English.
- Keep view models separate from backend DTOs when that improves clarity.
- Design for a structured RPG interface, not a generic chat.
- Avoid implementing gameplay authority in the frontend.

## Checklist

- Narrative panel exists or is planned.
- Action menu and action input are clear.
- Character sheet, inventory, quest log, companion panel, world state panel, Rune panel, and Lyra assistant panel have clear responsibilities.
- Components are small and testable.
- API calls are isolated in services.
- UI copy is professional and in English.

## Expected Output Format

- UI structure summary.
- Component/service list.
- State and API boundaries.
- Tests or checks.
- Documentation impact.
