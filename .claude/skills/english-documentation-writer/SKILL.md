# English Documentation Writer

## Purpose

Write README files, docs, ADRs, roadmap entries, and technical documentation in professional English.

## When To Use It

Use this skill for all public documentation and repository-facing explanations.

## Rules

- Use English only.
- Do not use casual Italian.
- Avoid AI-sounding filler and exaggerated claims.
- Avoid vulgar, adult, or portfolio-risky framing.
- Prefer clear software-project language.
- Explain technical choices like a real engineering project.
- Describe relationship features as character-driven storytelling or companion progression.

## Checklist

- Is the text accurate to the current implementation status?
- Is it useful to a developer or reviewer?
- Does it explain why the choice matters?
- Does it avoid overpromising?
- Is the tone professional and concise?

## Expected Output Format

- Title or heading.
- Short summary.
- Practical sections with clear bullets or tables when helpful.
- Links to related docs when useful.
