# Portfolio Polish

## Purpose

Keep the repository recruiter-friendly and professionally presented.

## When To Use It

Use this skill when editing README files, public docs, GitHub descriptions, issue templates, screenshots, release notes, or demo plans.

## Rules

- README must highlight architecture, local AI, deterministic game engine, memory, Rune, and Lyra.
- Avoid unprofessional wording.
- Describe relationship systems as character-driven storytelling, companion progression, or narrative relationship systems.
- Docs must explain why the project is technically valuable.
- Do not oversell unfinished features.

## Checklist

- Is the current project status honest?
- Are technical pillars easy to identify?
- Is public wording suitable for GitHub and recruiters?
- Are sensitive design topics framed professionally?
- Are docs linked and easy to navigate?

## Expected Output Format

- Public-facing summary.
- Strengths to highlight.
- Wording risks to remove.
- Suggested improvements.
- Commit message suggestion when relevant.
