# Project Brief

## Purpose

Preserve the identity and public positioning of **The World I Can Develop**, internally codenamed **Demon King**.

## When To Use It

Use this skill when writing public docs, READMEs, roadmap entries, issue descriptions, release notes, portfolio summaries, or feature briefs.

## Rules

- Public title: **The World I Can Develop**.
- Internal codename: **Demon King**.
- Genre: local-first AI-driven fantasy text RPG.
- Core mechanic: **Rune**, a divine pseudocode-based magic system.
- Divine assistant: **Lyra**.
- Present the project in professional, portfolio-safe language.
- Describe relationship features as narrative relationship systems, companion progression, or character-driven storytelling.
- Avoid private jokes, informal Italian, vulgar framing, or overclaiming unfinished features.

## Checklist

- Confirm the public title and codename are used correctly.
- Mention Rune and Lyra only in ways that match the current scope.
- Keep local-first AI and deterministic game rules visible.
- Make the text suitable for GitHub and recruiters.
- Avoid implying unfinished systems are already implemented.

## Expected Output Format

- Short project summary.
- Relevant technical pillars.
- Current implementation status.
- Clear next step or documentation link when useful.
