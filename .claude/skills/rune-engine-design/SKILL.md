# Rune Engine Design

## Purpose

Guide **Rune**, the divine pseudocode magic system.

## When To Use It

Use this skill when defining Rune syntax, validation, effects, costs, progression, UI, or backend execution.

## Rules

- Rune must be powerful but bounded.
- Validate syntax, cost, level, complexity, target, and allowed effect.
- MVP must start with minimal commands.
- Advanced reality modification is post-MVP.
- Backend validates all Rune effects.
- Lyra may explain Rune errors, but cannot approve effects.

## Checklist

- Is the command part of the MVP command set?
- Is syntax validation defined?
- Are target and effect rules clear?
- Is the cost or resource impact defined?
- Is level or progression gating needed?
- Is error feedback clear enough for Lyra to explain?

## Expected Output Format

- Rune command or feature summary.
- Validation rules.
- Allowed effects.
- Failure cases.
- Tests/checks.
- Documentation update.
