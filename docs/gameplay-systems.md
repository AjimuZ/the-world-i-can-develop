# Gameplay Systems

The game should feel like a text RPG product rather than a simple chat.

## Core Loop

1. The player reads the current narrative and world state.
2. The player chooses or writes an action.
3. The backend validates the action against deterministic state.
4. The AI generates narrative feedback when needed.
5. The backend validates any proposed state changes.
6. The game saves updated state, events, and summaries.

## Planned Systems

- Narrative panel for story text and consequences.
- Action menu and free action input.
- Character sheet for stats, progression, status, and Rune mastery.
- Inventory for items, equipment, consumables, and key objects.
- Quest log for active objectives and completed milestones.
- Companion panel for party members and character-driven storytelling.
- World state panel for location, factions, known events, and active conditions.
- Rune panel for divine pseudocode commands.
- Lyra assistant panel for guidance, analysis, and explanations.

## Relationship Design

Relationship features should be described publicly as narrative relationship systems, companion progression, and character-driven storytelling. The tone should stay mature, elegant, and portfolio-safe.
