# Learning Path

Each phase should teach one or two concrete engineering concepts.

| Phase | Learning Focus |
| --- | --- |
| Repository setup | Documentation structure, development workflow, Claude Code configuration. |
| Angular shell | Component boundaries, TypeScript models, UI state flow. |
| Spring Boot shell | REST controllers, service boundaries, health checks. |
| Docker Compose | Local infrastructure, PostgreSQL configuration, repeatable setup. |
| Game models | Domain modeling, persistence boundaries, migrations. |
| New game flow | End-to-end frontend/backend integration. |
| Narrative loop | Adapter design and AI boundary control. |
| State validation | Trust boundaries and deterministic rule enforcement. |
| Rune MVP | Parsing, validation, error reporting, gameplay balancing. |
| Lyra MVP | Assistant UX, explanations, contextual guidance. |
| Memory | Event logs, summaries, retrieval discipline. |
| Portfolio polish | Public communication, architecture documentation, demo readiness. |
